CONCURRENCY PATTERNS CHEATSHEET

WORKER POOL

Producer
   |
Jobs Channel
   |
Workers

Used when concurrency must be bounded.

Examples:
- API calls
- DB operations
- Queue processing

--------------------------------------------------

FAN-OUT

One producer
Many workers

Goal:
Increase throughput.

--------------------------------------------------

FAN-IN

Many producers
One result stream

Goal:
Aggregate results.

--------------------------------------------------

PIPELINE

Stage1 -> Stage2 -> Stage3

Each stage:
- Receives input
- Processes
- Produces output

--------------------------------------------------

PIPELINE BENEFITS

- Separation of concerns
- Parallelism
- Reusability
